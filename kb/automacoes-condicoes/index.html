Lógica Condicional em Formulários e Pesquisas

A Lógica Condicional permite criar formulários e pesquisas dinâmicas. Com ela, é possível mostrar ou ocultar campos, redirecionar usuários, exibir mensagens, desqualificar leads ou pular para diferentes slides, tudo com base nas respostas fornecidas.

Este guia explica como usar a Lógica Condicional, as novidades da versão 2 e quais campos são suportados.

O que é Lógica Condicional

A Lógica Condicional automatiza ações dentro de um formulário ou pesquisa com base nas respostas do usuário.

Você pode definir condições como: - Se a resposta for "Não", mostrar uma mensagem. - Se a pontuação for abaixo de 5, desqualificar o lead. - Se a data estiver dentro de um intervalo, redirecionar para outra página. - Em pesquisas, pular para um slide específico em vez de mostrar todas as perguntas.

Isso cria uma experiência mais personalizada e eficiente para os respondentes.

Novidades da Versão 2

  • Construtor Simplificado em 3 Passos: Selecione o Campo → Selecione o Estado → Forneça um Valor
  • Operador "Entre": Para campos numéricos, de data, pontuação e monetários
  • Ramificação de Pesquisa (Pular Para): Crie saltos de slide para slide
  • Suporte Expandido a Campos: Caixas de seleção, multi-dropdowns, Termos e Condições 1 e 2, uploads de arquivos e assinaturas
  • Novos Alvos de Mostrar/Ocultar: Elementos de pagamento, texto, HTML, imagem e Termos e Condições
  • Filtragem de Lista de Regras (Pesquisas): Filtre regras por campo ou slide
  • Proteção de Ciclo: Impede loops (Slide A → Slide B → Slide A)
  • Localizado e Acessível: Interface traduzida com navegação por teclado

Ações Disponíveis

Ao abrir a Lógica Condicional, você verá quatro tipos de ações que pode criar:

Ação Descrição
Redirecionar para URL Envia os usuários para uma página da web específica após o envio
Exibir Mensagem Mostra uma mensagem personalizada com base nas respostas
Desqualificar Lead Rejeita automaticamente envios de formulário não qualificados
Mostrar/Ocultar Campos Controla dinamicamente quais campos aparecem ou são ocultados

Todas as condições são executadas na ordem de cima para baixo. Regras correspondentes posteriores substituem as anteriores, quando aplicável.

Como Configurar a Lógica Condicional

Você pode adicionar e gerenciar regras diretamente no Construtor de Formulários ou Pesquisas.

  1. Abrir Lógica Condicional
  2. No construtor, clique em Lógica Condicional na barra superior.
  3. Para pesquisas, você também pode abri-lo a partir do alerta da barra lateral de um slide.
  4. Adicionar uma Nova Condição
  5. Clique em Adicionar Nova Condição.
  6. Escolha uma ação (Redirecionar, Mensagem, Desqualificar ou Mostrar/Ocultar).
  7. O construtor de lógica será aberto para essa ação.
  8. Construir a Regra
  9. Siga o fluxo guiado de 3 passos:
    • Selecionar Campo: Escolha o campo do formulário ou slide para basear a regra.
    • Selecionar Estado do Campo: Escolha a condição (por exemplo, é igual a, está vazio).
    • Fornecer um Valor: Insira o valor correspondente (texto, número ou data).
  10. Você pode adicionar condições adicionais usando os conectores E ou OU.
  11. Alguns operadores não exigem valores, como "está vazio" ou "está marcado".
  12. Escolher o Resultado da Ação
  13. Dependendo do tipo de ação:
    • Redirecionar: Adicione um URL válido (por exemplo, https://exemplo.com).
    • Exibir Mensagem: Insira o texto da sua mensagem.
    • Mostrar/Ocultar Campos: Escolha quais campos aparecem ou são ocultados.
    • Desqualificar Lead: O formulário interrompe o envio quando a condição é atendida.
    • Pular Para (Apenas Pesquisas): Escolha o slide ou pergunta de destino.
  14. Salvar
  15. O botão Salvar é ativado somente após todas as entradas necessárias serem válidas.
  16. O sistema verifica se há valores ausentes e impede loops automaticamente.

As regras são executadas de cima para baixo, e as correspondências posteriores substituem as anteriores.

Usando "Se/Senão" em Automações

O "Se/Senão" permite criar Automações avançadas para executar diferentes ações com base em condições. Ele usa lógica condicional: se A acontece no gatilho, então faça X. Se B acontece, então faça Y, e assim por diante.

Novas Funcionalidades

  1. Duplicar Branch:
    • Economize tempo copiando branches existentes.
    • Clique no menu de três pontos em um branch e escolha "Duplicar Branch".
    • O branch copiado aparecerá abaixo do original com "(cópia)" adicionado ao nome. Renomeie se necessário.
  2. Reordenar Branches:
    • Organize visualmente sua Automação arrastando e soltando branches.
    • Clique no ícone "Reordenar Branches" para reordená-los.
    • Não é possível reordenar o branch "Nenhum", mas você pode mover todos os outros.
  3. Renomear Branch "Nenhum":
    • Agora você pode nomear o branch "Nenhum" para melhor organização.
    • Clique no nome do branch e digite o nome desejado.
    • Certifique-se de inserir um nome, pois não pode ser deixado em branco.

Como Duplicar um Branch

  1. Dentro da Automação, localize o branch que deseja duplicar.
  2. Clique no menu de três pontos (reticências) ao lado do nome do branch.
  3. Selecione a opção "Duplicar Branch".
  4. O branch duplicado aparecerá abaixo do original, com "(cópia)" adicionado ao nome.
  5. Clique no nome do branch duplicado para renomeá-lo, se necessário.

Como Reordenar Branches

  1. Dentro da Automação, localize o ícone "Reordenar Branches".
  2. Clique no ícone "Reordenar Branches" para ativar o modo de reordenação.
  3. Arraste e solte os branches na ordem desejada.
  4. O branch "Nenhum" não pode ser movido.

Como Renomear o Branch "Nenhum"

  1. Dentro da Automação, localize o branch nomeado como "Nenhum".
  2. Clique no nome do branch ("Nenhum").
  3. Digite o novo nome desejado para o branch.
  4. Certifique-se de inserir um nome válido, pois o campo não pode ficar vazio.

Condições If/Else em Automações

Este artigo explica como usar condições If/Else em Automações, incluindo operadores de comparação de tempo e solução de problemas com condições AND/OR.

Como usar condições AND/OR

Ao criar condições em Automações, você pode usar "Inclui" ou "Não inclui" para campos de múltipla escolha, como tags, caixas de seleção e dropdowns.

Condição AND

Para criar uma condição "E", use a opção "Não inclui" para garantir que um contato NÃO tenha ambas as tags especificadas. Se o contato tiver UMA das tags, a condição falhará.

Exemplo: Verificar se um contato NÃO possui as tags 'consulta_agendada' E 'consulta_confirmada'.

Condição OR

Para criar uma condição "OU", divida as tags em condições separadas e escolha a opção "OU". Se o contato atender a UMA ou AMBAS as condições, a condição será verdadeira.

Exemplo: Verificar se um contato possui a tag 'tag_A' OU a tag 'tag_B'.

Ramificações If/Else

As ramificações If/Else suportam até 10 resultados diferentes por evento.

O que acontece quando duas ramificações são verdadeiras?

O sistema seguirá o primeiro caminho/ramificação correto na ordem em que você configurou as condições. A ordem é de cima para baixo.

Operadores de Comparação de Tempo

Os operadores de comparação de tempo comparam a data e hora atuais com um valor especificado. Eles usam unidades como dia da semana, dia do mês, mês, ano e hora.

Como usar

  1. Escolha o parâmetro de tempo desejado.
  2. Selecione o operador de comparação no menu dropdown.

Operadores

  • É: Verifica se a unidade de data selecionada é igual ao valor de entrada.
    • Exemplo: "Dia da Semana É Segunda-feira" (verdadeiro apenas se hoje for segunda-feira).
  • Não é: Verifica se a unidade de data selecionada é diferente do valor de entrada.
    • Exemplo: "Dia da Semana Não é Sábado" (verdadeiro para todos os dias, exceto sábado).
  • É Depois: Verifica se a unidade de data selecionada é posterior ao valor de entrada.
    • Para horas, considera minutos a partir da próxima hora.
    • Exemplo: "Hora Atual É Depois das 18h" (verdadeiro a partir das 19h).
  • É em ou Depois: Verifica se a unidade de data selecionada é igual ou posterior ao valor de entrada.
    • Para horas, considera minutos a partir da mesma hora.
    • Exemplo: "Hora Atual É em ou Depois das 18h" (verdadeiro a partir das 18h).
  • É Antes: Verifica se a unidade de data selecionada é anterior ao valor de entrada.
    • Exemplo: "Mês Atual É Antes de Junho" (verdadeiro para todos os meses antes de junho).
  • É em ou Antes: Verifica se a unidade de data selecionada é igual ou anterior ao valor de entrada.
    • Para horas, considera minutos a partir da mesma hora.
    • Exemplo: "Hora Atual É em ou Antes das 18h" (verdadeiro até as 18h59).
  • Não está vazio: Verifica se o campo tem um valor.
  • Está vazio: Verifica se o campo não tem valor.

Requisitos

  • Conhecimento básico de Automações na nossa plataforma.
